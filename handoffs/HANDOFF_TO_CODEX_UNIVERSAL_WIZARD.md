# HANDOFF_TO_CODEX_UNIVERSAL_WIZARD

> **Status:** READY for Codex to claim. Replaces "edit 4 config files by hand" with `npm run wizard` — one interactive command that picks the workspace, optionally links a GitHub repo, wires every supported MCP client, and verifies the result. Builds on the existing non-interactive `scripts/install-clients.mjs`; wizard is a thin TTY wrapper that drives the same code paths.

## 1. Mission

Make HermesProof installable on **any project** by a non-coder operator in under 90 seconds, without opening any config file by hand. The interactive wizard:

1. Detects the operator's platform (Windows / macOS / Linux) and tailors the prompts.
2. Asks for the **workspace path** (path completion + validation; offers to create the dir if missing).
3. Optionally asks for a **GitHub repo** (clone it locally, or link an existing local clone to its remote, or skip if local-only).
4. Detects which MCP clients are installed (Claude Desktop, Claude Code CLI, Codex CLI, Windsurf/Cascade, Cursor, VS Code Copilot agent mode, **Anthropic API console / Claude Agent SDK** if a key is present).
5. For each detected client, writes the `hermes3d-locks` config (with timestamped backups of any prior config), shows a one-line confirmation, and moves on.
6. Runs `hermes_doctor` against the new workspace and prints a green-or-red summary.
7. Bootstraps `.hermes3d_orchestrator/` via `init-project`.
8. Optionally runs `truth-gates --ci` and reports pass/fail.
9. Prints a copy-pasteable "first message to send any agent" that respects the AGENTS.md / standing-prompt convention.

**Friction removed:** no JSON / TOML / hooks-file editing by hand. No env-var guesswork. No "did I get the path escaping right on Windows" questions.

## 5. Required behavior

### 5.1 Entry point + flags

```bash
npm run wizard                                # full interactive
npm run wizard -- --dry-run                   # show what WOULD happen, no writes
npm run wizard -- --workspace <path>          # skip workspace prompt
npm run wizard -- --github <owner/repo>       # skip GitHub prompt; clone or link
npm run wizard -- --clients claude-code,codex # only wire these clients (skip detection)
npm run wizard -- --no-truth-gates            # skip the post-install verification
npm run wizard -- --json                      # machine-readable output (for CI / outer wizards)
npm run wizard -- --yes                       # accept all defaults; non-interactive but still writes
```

### 5.3 Path validation rules

- Reject paths that are not absolute (after expansion of `~` and `%USERPROFILE%`).
- Reject paths containing newline / null / control chars.
- Reject paths whose resolved form ends in `.git/` (must be the workspace, not the .git dir).
- On Windows, accept both `\` and `/` separators in user input; normalize via `path.resolve`.
- On macOS/Linux, expand `~` to `$HOME`.
- If path doesn't exist, prompt before creating; default `N` (don't auto-create).

### 5.4 GitHub repo handling

- If `gh` CLI is on PATH and authed: use `gh repo clone` for clones (handles auth, redirects).
- Else use `git clone https://github.com/<owner>/<repo>` (will fail on private repos without HTTPS auth; surface that clearly).
- If user picks "link existing local clone to a different remote": offer `git remote set-url origin <new>` only if `--force` flag passed; else refuse and explain.
- Default behavior: do nothing if a `.git/` exists. The workspace doesn't need GitHub at all; this step is purely optional.

### 5.5 Anthropic / Claude Agent SDK integration (current versions)

For consumers using the Anthropic API directly (no Claude Desktop or CLI) — for example, in Claude Agent SDK programs:

- Detect `ANTHROPIC_API_KEY` in env (DO NOT print or log the key).
- Write a sample script at `<workspace>/.hermesproof/anthropic-sdk-example.mjs` showing how to call Claude Agent SDK with `hermes3d-locks` as an MCP server. Example template covers: 1) instantiate the agent, 2) attach the MCP server with `command: "node"` + path to `src/server.mjs`, 3) include a system prompt referencing `examples/QUEUE_DISCIPLINE.md` step 0.
- Do not write or read the API key value anywhere — the example script reads from `process.env.ANTHROPIC_API_KEY` at runtime.
- Reference the **current** Claude Agent SDK shape as of this brief's date: `@anthropic-ai/claude-agent-sdk` (verify the exact package + entry-point name when you implement; if the SDK rename or split, follow the upstream docs and note it in the example's header comment).

### 5.6 Per-client writer responsibilities

Each writer in `wizard-writers.mjs` exposes:

```js
async function detect()       // returns { detected: bool, version?: string, configPath?: string }
async function backup(path)   // returns the backup path string (timestamped)
async function write(opts)    // writes config; opts = { workspaceRoot, serverEntry, owner? }
async function verify()       // runs the client-specific verification (e.g. `claude mcp list`)
```

Reuse logic from existing `scripts/install-clients.mjs`. Refactor it to expose programmatic functions per client; the existing CLI flag interface stays as a thin shell over the new API.

## 12. Hard rules

- DO NOT print, log, echo, or write any value, prefix, or suffix from `ANTHROPIC_API_KEY`, `CODERABBIT_API_KEY`, `GITHUB_TOKEN`, `MINIMAX_API_KEY`, `DEEPSEEK_API_KEY`, `HF_TOKEN`, `AZURE_SPEECH_KEY`, `SILICONFLOW_API_KEY`, or any other key in `.env`. The wizard reads these only via `process.env.<NAME>` at runtime, only when needed, and only enough to detect presence (not value).
- DO NOT add new MCP tools or change the server.
- DO NOT add new dependencies. Stay on stdlib (`node:readline`, `node:fs`, `node:os`, `node:path`, `node:child_process`). The TTY prompts use stdlib only — no `inquirer`, no `prompts`, no `enquirer`.
- DO NOT remove the existing manual SETUP_*.md guides. The wizard is the recommended path; the manual guides remain reference.
- DO NOT clobber an existing client config without first writing a timestamped backup.
- DO NOT auto-run `truth-gates` if the user passed `--no-truth-gates`.
- The wizard output, on success or failure, MUST never include any portion of any secret. Sub-agent spawn (if you split work): declare WRITE SCOPE explicitly per `docs/PARALLEL_SUBAGENT_DISCIPLINE.md`.
