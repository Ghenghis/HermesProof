## ✦ Net-new from Codex's adversarial pass (Claude missed)

These weren't on Claude's radar — Codex's adversarial framing surfaced them:

1. **P0-2: `writeJsonAtomic` temp-name collisions** — Codex actually ran a 50-parallel temp probe and reproduced ENOENT rename failures because `${pid}.${Date.now()}.tmp` isn't unique enough at high concurrency. This is the **foundational helper** under every v0.7 store; fixing the per-module mutex (P0-1) without fixing this leaves writes unsafe.
2. **P0-3: real existing chain break at `ev_47347bd34f9a282d`** — Codex ran `hermes_verify_evidence`, found a broken hash chain. Claude predicted this could happen; Codex showed it already did.
3. **P1-9: `tests.unit` truth-gate runs only 5 of 12 test files** — Claude noticed coverage gaps in test files, but Codex spotted that the *proof harness itself* claims "full unit suite" while running a subset. The proof artifact lies about its own coverage.
4. **P1-11: legacy task/handoff ID path traversal in lock-manager** — Claude focused on v0.7 sinks; Codex audited the v0.5/v0.6 sinks too and found length-only validation.
5. **P1-14: `hermes_list_agents` omits anonymous role state** — Codex traced the data flow tighter and saw the route description doesn't match the implementation.
6. **P1-16: Hermes Agent decisions promised but not evidenced** — bridge says decisions are evidenced; no append code path exists.
7. **P2-17: gate-runner env+output secret leakage** — gates inherit parent env; output tail goes into the append-only ledger. Claude touched this generally (#14); Codex traced the specific path.
8. **P2-20: CI MCP-spec probe uses `2024-11-05`** — every doc claims `2025-11-25`, but `truth-gates.mjs:1301` handshake probe uses the older version. Even if docs were synced, CI wouldn't be proving them.
9. **P2-23: event-flow SVG 13 vs implementation 15** — fine-grained diagram drift Claude didn't catch.

## ✦ Recommended fix sequencing (post-Codex revision)

**P0 immediate — single coordinated PR ("v0.7 hardening pass"):**
1. **Fix `writeJsonAtomic`** — make temp-name truly unique (`crypto.randomBytes(8).toString("hex")`). Mandatory pre-req for the mutex work.
2. **Extract shared `Mutex` / serialized JSON-store helper** — apply to reputation/skill-rotation/anonymous-orchestrator/lock-manager `heartbeat`/queue `heartbeat`/event-manager. Concurrency tests modeled on `a2a-stub.test.mjs`.
3. **Extract `EvidenceLedger` core module** — single chained ledger; route v0.7 evidence through it. Migrate existing `anonymous-orchestrator/evidence.ndjson` entries.
4. **Investigate + repair the existing chain break at `ev_47347bd34f9a282d`** — quarantine, verify nothing depends on the broken segment, decide between rebuild-from-checkpoint vs accept-and-document.
5. **Harden `hermes_user_grant_session`** — require `scope` non-null/non-empty; add server-side allowlist for `granted_by:"human"`; require shared-secret env binding.
6. **Harden `hermes_agent_request_user_session`** — require `HERMES_AGENT_SCOPE` set; intersect requested with hard ceiling; optional signature verification on provider response.

**P1 next minor — separate PRs:**
7. 1-line `_resolvedProviders` fix → use `this._mergedProviders[name]`. Add registry-provider unit test.
8. Install `server.mjs` shutdown handlers, drain in-flight chains.
9. Make truth-gate `tests.unit` run the full `npm test` manifest (single source of truth).
10. New `scripts/v07-stdio-roundtrip-smoke-test.mjs` — round-trip every v0.7 tool.
11. Add path normalization to lock-manager legacy ID sinks.
12. Doc-pass refresh: gate count 29 + tool list completeness everywhere; build `expectedTools` from `src/server.mjs` at gate-run time.
13. Fix `hermes_list_agents` to include anonymous role state.
14. Wire Hermes Agent decision evidence-write path that the docstring promises.

**P2 / P3:** triage during sprint planning; many are 1-3 line fixes (annotations, naming, magic numbers, SVG text patches).
